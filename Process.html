Process

AODA compliance & Accessibility

The website has passed all the contrast tests in WCAG AA, AAA in the normal, large and graphical objects. As the contrast radio of the website is 14.61:1. The website fits the contrast ratio of 4.5:1 for normal text and 3:1 for large text in WCAG 2.0., reach the contrast radio of 3:1 for graphics and user interface components (such as form input borders) in WCAG2.1, and fulfill the contrast ratio of 7:1 for normal text and 4.5:1 for large text in WCAG Level AAA.

In the part of accessibility, all the elements will include the labels such as alternative text in all media and description to improve a better net surfing experience to the visual impairment users. The website has a clean and organized code structure and content as the website fulfill the WCAG 2.0 and ATAG 2.0 principles, and every element in the website are named and provide assistance to operate properly in order for the users can have a great perceivable, operable, understandable and robust experiences when users surf the website.

I went through a hard time to resolve the problem of creating the images into a lightbox image (Clickable image that allows zooming to the original size of the picture). There are 2 modal buttons in my project. The first one is the lightbox image on the production page, and the second one is the modal button on the form page. It is hard for me because I didn't notice there are 2 models that have the same id/variable in the custom.css file and the boostrap.css file before I searched the result up on the browser. This indicates only one of the modal work at the time. As the modal button worked at that time, but the modal image was not be functioning since the code is overlapping. After I looked it up, I realize I have to edit one of the variables from one of the modals in order to get the modal image in function, and after multiple attempts, both of them work properly now.
